데이터 모델링
- 개념설계
   ) 개체의 관계를 나타낸 것
- 논리설계

관계형 데이터모델
- 현재 가장 많이 사용되는 데이터베이스 모델
- 데이터가 최소한의 의미를 가지는 테이블 형태들로 표현
- 사용자가 데이터를 쉽게 다룰 수 있도록 해주는 질의어(SQL) 제공
- SQL(Selected Query Language)
  > 자연어에 가까운 문법
  > 어떻게(how) 가 아닌 어떤(what) 데이터를 작업
  > 표준화 된 작업

도메인
논리 - 중복이 제거된 값

릴레이션(relation) ; 테이블
- 현실세계의 개체(물리설계)
- 동일한 데이터베이스 내에서는 같은 이름을 갖는 릴레이션이 존재할 수 없음

튜블(tuple)
- 개체에 속한 구성원들 개개의 정보
- 한 릴레이션에 속한 튜플의 개수(카디널리티; cardinality)는 변할 수 있음
- 인스턴스와 유사하나 실무에서 사용되는 빈도 적음

속성(attribute)
- 차수(degree) : 속성의 개수
- 릴레이션에서 관리하는 구체적인 정보 항목
- 관리해야 할 필요가 있는 속성들만을 선택, 릴레이션에 포함
- 고유한 이름은 가져 동일 릴레이션 내에서 같은 이름의 속성이 존재할 수 없음

도메인(Domain)
- 각각의 속성들이 가질 수 있는 값들의 집합

키의 필요성
- 기본키
   ) 후보키 중 식별하는 데 기준으로 사용할 키
   ) Not Null, Not Duplicate
- 대체키
   ) 후보키 중 기본키로 선택되지 않은 나머지 키
   ) 0개 이상
- 후보키
   ) 


후보키가 될 수 있는 조건
1 유일성(중복x)
2 not null(값이 없는 경우가 아니어야 함)
3 자주 사용되며 중요도가 높은 키

후보키 중 기본키?
1 해당 테이블을 대표
2 데이터 길이가 짧은 것
3 사용 빈도가 높은 컬럼

- 복합키
   ) 하나의 컬럼이 후보키의 역할을 하지 못하고
     두 개 이상의 컬럼이 합쳐져야 후보키의 역할을 하는 경우
- 외래키(Foreign Key)
   ) 
   ) 자식 테이블의 외래키를 이용해 부모 테이블의  기본키로 사용하는 경우 '참조'라고 한다
     자식 --- 참조 --> 부모
   ) 부모 테이블의 기본키를 자식 테이블의 외래키로 적용시키는 경우 '전이'라고 한다.
     자식 <-- 전이 --- 부모
   ) 부모의 기본키는 하나만 존재 가능하지만 자식의 외래키는 여러개 존재할 수 있다.
   )
